Day Tradin Bot and Machine Learning

Created multiple trading bots based on popular trading strategies. Then trying to use Machine Learning to predict if the trade was going to be a win or a loss.

Technology Stack:

  • Python
  • Tableau
  • MetaTrader API
  • scikit-learn

View on GitHub:

Example MACD Buy Strategy

One example of a bot I developed is based on the popular MACD crossover trading strategy. This strategy is a rules-based system. When the price surpasses the exponential moving average and the MACD signal line crosses the center line, a buy is triggered using the MetaTrader python API.

Simulation to Reality Inference

I conducted a backtest using the MACD bot on five years' worth of US30 data. Additionally, I performed a detailed analysis using Tableau to observe the bot's trading profitability and patterns. This rigorous backtesting and data gathering process was applied to every trading strategy, laying the foundation for my machine learning model.

Model Performance

The dataset, which includes Open, Close, High, and Low values, covers around 900 individual trades. I used the XGBoost machine learning model for this analysis because it works well with tabular data and can handle imbalanced datasets. The trading strategy had more losses than wins, leading to an imbalanced data set.

There's a clear difference in performance between the training and test sets, hinting at possible overfitting. I believe this overfitting is due to the limited size of our dataset. If we had access to more comprehensive data for Open, Close, Low, and High values, we could improve our model through more extensive backtesting.
